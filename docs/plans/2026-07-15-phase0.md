Kafka(KRaft)、Testcontainers、GitHub Actions、Terraform、GKE Autopilot。

## Task 2:本地環境——docker-compose + Flyway + MyBatis(PR #2)

**概念講解(導讀會展開,Kafka 從零講)**:
- **Kafka 是什麼**:分散式 commit log。producer 把訊息 append 到 **topic**,
  topic 切成多個 **partition**(平行度與順序的單位),consumer 以
  **consumer group** 分工消費、用 **offset** 記讀到哪。跟 RabbitMQ 那種
  「讀完即刪」的 queue 不同,Kafka 的訊息留著,consumer 自己記進度——
  這是重放(replay)與 effectively-once 設計的基礎。
- **KRaft**:Kafka 自帶的共識機制,取代舊時代的 ZooKeeper 依賴,
  單節點起最省事,本地開發標配。Phase 0 只讓它「在場」,程式碼還不碰它。
- **Flyway**:資料庫 schema 的版本控制。`V1__init.sql` 這種檔名即版本,
  啟動時自動把 DB 補到最新版,並在 `flyway_schema_history` 表記帳。

**Files**:
- Create: `docker-compose.yml`
- Create: `app/wallet/src/main/resources/application.yml`、`application-local.yml`
- Create: `app/wallet/src/main/resources/db/migration/V1__init.sql`
- Create: `app/wallet/src/main/java/tw/ewallet/wallet/ping/internal/SmokeMapper.java`
- Test: `app/wallet/src/test/java/tw/ewallet/wallet/SchemaSmokeIT.java`
