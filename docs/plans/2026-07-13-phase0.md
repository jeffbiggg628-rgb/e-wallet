# Phase 0 Implementation Plan ŌĆö Skeleton & Infrastructure

> **Execution model (differs from standard plans):** Jeff implements every step himself ŌĆö
> this is a Go-learning project. Steps therefore specify the goal, requirements,
> reference docs, and verification, but intentionally contain **no ready-to-paste
> implementations**. When stuck, ask Claude for a hint before asking for the answer.
> Claude reviews each task's PR (per the CLAUDE.md checklist) before it merges.

**Goal:** A running project skeleton: single-module Go monorepo, buf-managed proto
contract, local MySQL + Pub/Sub emulator via docker-compose, migration toolchain,
CI (lint / generate-check / test / build-push), minimal-cost Terraform GCP baseline,
and one verified build ŌåÆ push ŌåÆ deploy ŌåÆ destroy cycle on GKE Autopilot.

**Spec:** `docs/knowledge/discussions/2026-07-13-phase0-design.md`
**Cadence:** One task = one branch = one PR. CI green + Claude review before merge.
